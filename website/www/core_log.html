CORE FUNCTIONS: LOG

This package exports several entry points into the JS environment, including:

  • log
  • log0
  • log1
  • log2
  • debug
  • debug
  • warning
  • error
  • fatal
  • panic
  • withFields
  • setLevel
  • jsonFormatter

This API allows resource handlers to log.

LOG

log(message);

Logs a message in the default format

Example:


  log("hello");

LOG

log(message);

Logs a message in the default format

Example:


  log("hello");

LOG0

log0(message);

Logs a more important message.

Example:


  log0("hello");

LOG1

log1(message);

Logs a less important message

Example:


  log1("hello");

LOG2

log1(message);

Logs an even less important message

Example:


  log2("hello");

setLevel

setLevel(logLevel);

Only log messages at this severity or above. Values may be:

  • "debug"
  • "info"
  • "warning"
  • "error"
  • "fatal"
  • "panic"

jsonFormatter

log.jsonFormatter(format);

Instead of strings, the log will emit... JSON!

The format argument may be omitted, but if set, should be a string as documented here

debug

log.debug(message);

Log a message at debug level.

info

log.info(message);

Log a message at info level.

warn

log.warn(message);

Log a message at warning level.

error

log.error(message);

Log a message at error level.

fatal

log.fatal(message);

Log a message at fatal level.

panic

log.panic(message);

Log a message at panic level.

withFields

log.withFields(fieldObj);

Create a set of fields to be logged with a message. Returns an object with .debug(), .info(), etc, function properties.

For example:

log.withFields({omg: "Yeah!"}).info("this is awesome");

For more information see the logrus docs.
